# PRD: Чек-лист по действующим договорам с клиентами

## 1. Цель и проблема

Компания заключает договоры с клиентами, содержащие разрозненные и неунифицированные условия: суммы, порядок и сроки оплаты, перечень документов для оплаты, гарантийные обязательства, штрафные санкции, условия расторжения и прочие существенные положения. Отсутствие единого инструмента для фиксации и проверки этих условий приводит к риску пропуска важных пунктов договора, задержкам оплат и спорам с контрагентами.

**Цель продукта:** дать ответственным сотрудникам единый, простой в заполнении инструмент (файл Excel), который позволяет:
- фиксировать ключевые условия по каждому действующему договору в одном реестре;
- проверять полноту и корректность условий конкретного договора по стандартизированному чек-листу.

## 2. Целевые пользователи

- Менеджеры по работе с клиентами / аккаунт-менеджеры — вносят и обновляют данные по своим договорам.
- Бухгалтерия / финансовый отдел — используют данные об условиях и сроках оплаты, перечне документов.
- Юридический отдел — проверяет гарантийные обязательства, штрафные санкции, условия расторжения и иные положения.
- Руководитель / собственник процесса — получает сводный обзор по всем договорам.

## 3. Формат продукта

Файл Microsoft Excel (.xlsx) из двух листов:

1. **«Чек-лист по договорам»** — сводный реестр всех действующих договоров.
2. **«Форма проверки договора»** — детальная форма проверки одного договора с подсказками по каждому разделу.

## 4. Функциональные требования

### 4.1. Лист «Чек-лист по договорам» (реестр)

Таблица с полями (столбцами) на каждый договор:

| № | Поле | Описание |
|---|------|----------|
| 1 | № п/п | Порядковый номер записи |
| 2 | Контрагент (клиент) | Наименование клиента |
| 3 | № и дата договора | Реквизиты договора |
| 4 | Ответственный менеджер | ФИО сотрудника, ведущего договор |
| 5 | Сумма договора, с НДС | Итоговая сумма по договору |
| 6 | Валюта | Валюта расчётов |
| 7 | Условия оплаты | Предоплата / постоплата / поэтапная оплата |
| 8 | Срок оплаты | Количество дней или конкретная дата |
| 9 | Документы, необходимые для оплаты | Счёт, УПД/акт, накладная и т.д. |
| 10 | Гарантийные обязательства | Срок и условия гарантии |
| 11 | Штрафные санкции | Пеня, неустойка, процент и основание начисления |
| 12 | Условия расторжения | Срок уведомления, основания расторжения |
| 13 | Иные существенные условия | Конфиденциальность, форс-мажор, подсудность и т.п. |
| 14 | Статус проверки | Выпадающий список: Да / Нет / Частично / Уточняется |
| 15 | Примечание | Свободный комментарий |

**Требования к реализации:**
- Строка заголовка закреплена (freeze panes), выделена цветом, текст переносится по словам.
- Столбец «Статус проверки» — выпадающий список с фиксированными значениями (Да, Нет, Частично, Уточняется), реализован через data validation.
- Количество пустых строк в реестре — гибко настраиваемое, по умолчанию минимально (2 строки), пользователь дополняет по необходимости, копируя формат.
- Единый профессиональный шрифт (Arial) и согласованное оформление (заливка заголовка, границы ячеек, перенос текста).

### 4.2. Лист «Форма проверки договора»

Детальная форма для проверки одного конкретного договора, содержащая:
- Поля «Контрагент» и «№ и дата договора» для идентификации проверяемого документа.
- Таблицу с разделами проверки и подсказками, на что обратить внимание:

| Раздел | На что обратить внимание |
|---|---|
| Сумма договора | Соответствие итоговой суммы спецификации/приложениям |
| Условия оплаты | Предоплата / постоплата / поэтапно, проценты по этапам |
| Срок оплаты | Конкретный срок — количество дней с даты счёта/акта или точная дата |
| Документы для оплаты | Счёт, счёт-фактура/УПД, акт выполненных работ, накладная и иные |
| Гарантийные обязательства | Срок гарантии, порядок предъявления претензий, гарантийное обслуживание |
| Штрафные санкции | Пени/неустойка за просрочку оплаты и/или поставки, предельный размер санкций |
| Условия расторжения | Основания расторжения, срок уведомления, порядок возврата средств |
| Иные положения | Форс-мажор, конфиденциальность, подсудность/арбитраж, порядок изменения договора |

- Каждый раздел сопровождается столбцами «Статус» (выпадающий список Да/Нет/Частично/Уточняется) и «Комментарий».

## 5. Нефункциональные требования

- Файл открывается и корректно отображается в Microsoft Excel и LibreOffice Calc.
- Отсутствие ошибок формул (#REF!, #DIV/0! и т.д.) — на текущем этапе формулы не используются.
- Единообразное оформление: цветовая палитра, шрифты, ширина столбцов, перенос текста.
- Возможность расширения количества строк реестра без потери форматирования.

## 6. Пользовательские сценарии (use cases)

1. **Внесение нового договора.** Менеджер добавляет строку в реестр, заполняет поля 2–13, при необходимости прикладывает примечание.
2. **Проверка договора.** Ответственный сотрудник использует лист «Форма проверки договора» для пошаговой сверки условий по каждому разделу, выставляет статус и комментарий.
3. **Контроль статуса.** Руководитель просматривает столбец «Статус проверки» в реестре, чтобы увидеть, какие договоры полностью проверены, а какие требуют уточнения.
4. **Автоматизированная проверка (перспектива).** Пользователь загружает файл договора (PDF/DOCX), Claude анализирует текст и сверяет его с пунктами чек-листа, заполняя реестр/форму проверки автоматически.

## 7. Ограничения и допущения

- На момент составления документа реальные договоры пользователем не загружены — чек-лист является шаблоном без данных.
- Заполнение данными по конкретным договорам — отдельная задача, требующая загрузки документов (PDF, DOCX, XLSX, изображения) для последующего анализа.

## 8. Дальнейшие шаги

- Загрузка реальных договоров для автоматического заполнения реестра.
- При необходимости — добавление сводных показателей (общая сумма по всем договорам, количество договоров с просроченной проверкой и т.п.).
- При масштабировании — перенос реестра в общую систему (например, таблицу в облаке) для совместной работы нескольких пользователей.
